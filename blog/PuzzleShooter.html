Configuring the Client (Unreal Engine)

The client makes http requests to the REST API, which has various resources for various AWS Lambda functions. The client uses a combination of the responses returned from the API to join an Unreal Engine game server process hosted on an EC2 instance on a fleet. In the process, multiple clients interact with the main menu UI in order to start and stop matchmaking tickets. If the GameLift matchmaking configuration is able to make a potential match with the existing active matchmaking requests, a match will be made, and GameLift will try to find an available server process to host the game session on. Then, finally, the clients in that match will get an update through their matchmaking tickets with the necessary information to connect to the server.
